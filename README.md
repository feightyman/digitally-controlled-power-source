# 🔌 数控电源 (Digitally Controlled Power Source)

本项目是一个基于 STM32 微控制器开发的**数控电源 (DC Power Supply)** 固件程序。项目通过微控制器输出高精度的模拟基准电压来控制电源电路，同时通过内部 ADC 闭环采集实际电压与电流，并在 OLED 屏幕上提供良好的人机交互界面。

## 🛠️ 硬件架构

- **主控芯片**：STM32F103C6 (或同系列 STM32F1xx 芯片)
- **DAC (数模转换)**：MCP4725（12位高精度 I2C DAC），用于输出电压/电流调节的基准控制信号。
- **ADC (模数转换)**：使用 STM32 内置 ADC 采集后级电路的实际电压与电流反馈。
- **显示外设**：I2C OLED 屏幕（包含自带的数字与英文字库）。
- **人机交互**：独立按键 (Keys) 及 串口通信 (UART)。

## ⚙️ 核心技术实现 (Technical Implementation)

本项目在软件架构上进行了清晰的分层，通过 STM32CubeMX 生成底层的 HAL 库配置，并将核心业务逻辑与硬件驱动剥离，封装在独立的 `mycode` 目录下。以下是详细的技术实现细节：

### 1. 底层框架与外设配置 (HAL Layer)
- **HAL 库与时钟树**：采用 STM32 HAL 库开发，提供高可移植性。系统时钟及各类外设时钟在 `main.c` 和 `system_stm32f1xx.c` 中初始化。
- **外设应用**：
  - **ADC**：配置为多通道/轮询模式，对电源输出的真实电压（V_out）和电流（I_out）进行高频采样。
  - **UART**：开启接收中断机制，主要用于上位机/串口调试指令的下发。
  - **I2C**：系统配置了多路 I2C 或软件模拟 I2C 总线，挂载 OLED 与 MCP4725 芯片。

### 2. 硬件驱动中间件 (Driver / `mycode/`)
- **高精度电压控制 (`MCP4725.c / .h`)**
  - 利用 I2C 协议对 MCP4725 DAC 芯片进行写入。由于该芯片支持 12-bit 分辨率（0~4095），程序内部将用户设定的物理量（如 5.00V）通过换算公式映射为 DAC 的数字量。
  - 这个 DAC 输出的模拟电压，直接驱动外部线性稳压器（LDO）或开关电源电路的反馈引脚（FB），实现“数控”调节。
- **屏幕显示引擎 (`oled.c / .h / oledfont.h`)**
  - 实现了一套精简的 OLED 渲染引擎。底层通过 `IIC.c` 传输数据指令，支持点阵绘制、光标定位以及数值刷新。
  - 配合 `oledfont.h` 内部存储的字模点阵，能够实时在屏幕上刷新电源的“设定值(Set)”与“实际值(Real)”。
- **按键状态机 (`Key.c / .h`)**
  - 使用定时器或轮询方式进行按键的消抖（Debounce）处理。
  - 对用户输入进行特征捕捉（单击、长按步进等），防止因为机械按键抖动导致的电压设定值突变。

### 3. 核心业务逻辑与闭环控制 (`fun.c / main.c`)
- **主业务循环 (`fun.c` -> `fun()` 接口)**：
  - **数据融合与平滑处理**：对 ADC 采集到的海量原始数据进行软件滤波（均值滤波或滑动窗口），抑制电源纹波带来的跳跳字干扰，然后将其转换为实际的物理读数。
  - **响应与反馈更新**：将刷新后的反馈电压、电流值调用 `oled` 接口刷入屏幕。
- **异步串口命令交互**：
  - 定义了完整的状态标志位（如 `rx_flag`, `notarize_flag`, `change_flag`），实现了一个串口状态机。
  - UART 接收缓冲 (`rx_buf[1024]`) 配合索引 (`rx_index`) 处理动态长度的指令，允许用户通过串口控制终端实时监控电源状态并下发电压设定修改指令，完成远程控制。

### 4. DSP 与算力加速（可选）
- 引入了 `CMSIS DSP` 及 `NN` (神经网络) 等 ARM 官方底层加速库 (`arm_math.h`)。
- 虽然电源控制逻辑中主要进行代数运算，但 DSP 库能够为后续如果引入 **PID 数字闭环控制**（通过 `arm_pid_xxx` 函数）或进行高阶 IIR/FIR 滤波提供底层硬浮点和指令级加速支持。

## 📁 核心目录结构

```text
digitally-controlled-power-source/
├── Core/                      # STM32Cube 生成的核心初始化代码
│   ├── Inc/                   # main.h, gpio.h, usart.h, adc.h 等
│   └── Src/                   # main.c 主入口, STM32 HAL 初始化逻辑
├── Drivers/                   # ST 官方 HAL 库与 CMSIS 内核/DSP 支持库
├── MDK-ARM/                   # Keil uVision5 工程目录与编译产物
├── mycode/                    # 💡 核心用户业务代码
│   ├── fun.c / fun.h          # 综合业务逻辑、交互调度与数据处理
│   ├── IIC.c / IIC.h          # I2C 总线底层协议实现
│   ├── Key.c / Key.h          # 按键扫描与消抖逻辑
│   ├── MCP4725.c / MCP4725.h  # MCP4725 12位 DAC 驱动
│   ├── oled.c / oled.h        # OLED 屏幕驱动
│   ├── oledfont.h             # OLED 屏幕点阵字库
│   └── header.h               # 全局共享头文件
├── power.ioc                  # STM32CubeMX 工程配置文件
└── ...
```

## 🚀 编译与开发指南

1. **开发环境**：
   - 推荐使用 **Keil MDK (uVision 5)** 打开 `MDK-ARM/power.uvprojx` 进行编译。
   - 也可以通过导入 `.ioc` 文件在 STM32CubeIDE 中重新生成 Makefile 或 CMake 跨平台编译工程。
2. **硬件烧录**：
   - 编译无误后，使用 ST-Link 或 J-Link 通过 SWD 接口连接微控制器，点击 Keil 中的 Download 即可下载程序并仿真。
3. **引脚修改**：
   - 如果需要修改硬件连接（如更换 I2C 引脚或 ADC 通道），请使用 STM32CubeMX 打开 `power.ioc`，修改后点击 `Generate Code`，并同步更新 `mycode/` 目录下的底层映射。
